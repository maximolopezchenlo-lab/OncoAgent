Nobody likes a frozen screen! 🥶

When running heavy multi-agent triage via LangGraph on an AMD MI300X, the backend crunches a LOT of data. Previously, our UI would just "freeze" while waiting for the LLM to process medical guidelines, leading to timeouts.

We just optimized the OncoAgent dashboard by implementing generator streaming (`yield`) in Gradio. Now, the moment a doctor clicks "Send", the UI instantly acknowledges the request and streams loading states across the knowledge graph, NCCN evidence tabs, and the clinical report. Small code change, MASSIVE UX improvement! 🚀

2/ 🛠️ The secret? Sequence Packing + Unsloth.
Instead of feeding short medical texts one by one and wasting compute on padding tokens, we packed them perfectly into 2048-token sequences. The MI300X memory bandwidth handled the dense tensors flawlessly.

3/ 🚀 This changes our entire iteration speed. We can now run multi-epoch training on massive datasets multiple times a day. Next up: pushing the Tier 2 27B model to see how hard we can stress this GPU!
